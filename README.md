# PZK Site — Портал ПЦК

**PZK Site** - это информационный сайт Предметно-Цикловой комиссии технологии продуктов питания и естественно-научных дисциплин. Сайт разработан для удобного доступа студентов и преподавателей к расписанию, новостям, учебным программам и информации о педагогическом составе. Сайт является конкурсным.

## 📂 Структура файловой системы

```text
/
├── app/
│   ├── components/
│   │   └── MobileHeader.tsx    # Компонент мобильной навигации
│   ├── globals.css             # Глобальные стили, переменные и адаптивная верстка
│   ├── layout.tsx              # Корневой шаблон (Header, Footer, Metadata)
│   └── page.tsx                # Главная страница (Логика данных, расписание, модальные окна)
├── prisma/
│   ├── seed.ts                 # Функция для перевода базы данных из sqlite в postgresql
│   ├── schema.prisma           # Схема базы данных (News, Teacher, Activities)
│   └── dev.db                  # База данных для локального запуска
├── public/                     # Статические ресурсы (логотипы, изображения)
└── package.json                # Зависимости проекта (Next.js, Prisma, React)
```

## ⚙️ Функциональные возможности и логика

### 1. Обработка и форматирование данных (`page.tsx`)
* **MobileHeader:** Функция **`formatTeacherData`** отвечает за парсинг "сырого" текстового описания преподавателя из базы данных и преобразование его в структурированный вид путем очистки пробелов, переноса строк и сегментации. Функция находит в тексте ключевые фразы, разбивает массив данных на логические блоки и оборачивает их в соответствующие HTML-теги. Это позволяет хранить в БД простой текст, но выводить его как красиво оформленную анкету.
* **Управление состоянием:**
`useState`: Используется для управления модальными окнами (открытие/закрытие карточек преподавателей и новостей) и хранения загруженных списков.
`useEffect`: Отвечает за асинхронную загрузку данных при первом рендеринге страницы. Он инициирует запросы к API для получения новостей, списка преподавателей и данных о внеаудиторной работы.
* **Оптимизация через useMemo:** Для повышения производительности используется хук useMemo, который кэширует результаты вычислений. Это особенно важно при фильтрации данных: тяжелые операции по пересборке списков выполняются только тогда, когда меняются исходные данные, а не при каждом рендере страницы.
* **Обработка ошибок и состояний загрузки:** В коде предусмотрены проверки на наличие данных. Если список пуст или произошла ошибка при загрузке, интерфейс не «ломается», а выводит заглушки или уведомления (условный рендеринг).

### 2. Управление контентом (Prisma)
Сайт интегрирован с ORM Prisma для работы с таблицами:
* `News`: Хранение и отображение новостей ПЦК.
* `Teacher`: Информация о преподавателях и их квалификации.
* `Activities`: Данные о внеурочной работе студентов.

### 3. Навигация и адаптивность (`MobileHeader` & `Layout`)
* **MobileHeader:** Обеспечивает навигацию по разделам (Новости, Преподаватели, Программы, Расписание, Внеаудиторная работа, Контакты).
* **SEO-оптимизация:** В `layout.tsx` настроены метаданные (Title, Description) для корректного отображения сайта в поисковых системах.

### 4. Сложные механизмы стилизации (`globals.css`)
Файл стилей выходит за рамки простого оформления и реализует сложную архитектуру адаптивности и позиционирования:
* **Динамический расчет геометрии (`calc` & `rem`):** В коде активно используются прецизионные вычисления размеров, например: `height: calc(100% - 7.96875rem + 0.1875rem);`. Это позволяет создавать «живую» верстку, где высота боковых панелей или модальных окон автоматически подстраивается под высоту хедера, сохраняя идеальное выравнивание до пикселя вне зависимости от контента.
* **Математика адаптивности в Media Queries:** Вместо стандартных сеток, используется тонкая настройка через `@media (max-width: 47.9375rem)`. Использование дробных значений в `rem` для брейкпоинтов гарантирует отсутствие «мертвых зон» при масштабировании интерфейса и специфическую адаптацию под мобильные устройства с нестандартным разрешением.
* **Управление переполнением и мобильный UX:** Для компонента `.schedule-table` реализован механизм `overflow-x: auto` в сочетании с `white-space: nowrap`. Это превращает громоздкие таблицы расписания в удобные горизонтальные свайп-зоны на смартфонах, сохраняя читаемость данных без уменьшения шрифта.
* **Продвинутая верстка модальных окон:** Классы `.modal-content` и `.activity-modal-content` используют комбинацию абсолютного и фиксированного позиционирования с динамическим расчетом ширины: `width: calc(100% - 1.5rem)`. Это обеспечивает эффект «плавающего окна» с равномерными отступами от краев экрана, что сложно реализовать стандартными методами.
* **Контроль соотношения сторон (Aspect Ratio):** Для изображений в блоке новостей (`.news .block img`) применено свойство `aspect-ratio: 4 / 3`. Это критически важная деталь, которая предотвращает «прыжки» верстки (Layout Shift) при загрузке картинок, заранее резервируя правильное место под контент.

## 🚀 Как запустить локально (необходимо поменять базу данных на локальную в schema.prisma)

1.  Установите зависимости:
    ```bash
    npm install
    ```
2.  Настройте базу данных и выполните миграции Prisma:
    ```bash
    npx prisma generate
    npx prisma db push
    ```
3.  Запустите сервер для разработки:
    ```bash
    npm run dev
    ```
